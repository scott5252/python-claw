1. Write high-risk unit tests first for the shared observability contracts, covering required structured-event field population, stable `failure_category` mapping, `trace_id` and correlation propagation helpers, bounded redaction and preview behavior, and explicit prevention of secrets or high-cardinality identifiers leaking into the wrong telemetry surfaces.
2. Add high-risk API and authorization tests first for `GET /health/live`, `GET /health/ready`, and `/diagnostics/*`, proving liveness stays process-only, readiness is dependency-aware and fail-closed, diagnostics access is denied by default without valid admin or internal credentials, and responses stay bounded, paginated, and sanitized.
3. 5. Introduce the shared observability package under `src/observability/` with dependency-injected helpers for trace context, structured JSON logging, redaction, failure classification, health evaluation, diagnostics aggregation, metrics, and tracing, keeping local no-op or console behavior available when external exporters are not configured.
6. 7. Replace ad hoc or free-form logging across the gateway, worker, runtime, dispatcher, media, execution, and node-runner paths with the shared structured-event helpers so accepted requests, duplicate replays, claims, lease refreshes, retries, degraded continuity, node execution, attachment normalization, and outbound delivery attempts all emit machine-parseable events tied to persisted state transitions.
