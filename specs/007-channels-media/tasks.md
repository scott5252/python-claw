# Tasks 007: Channels, Streaming, Chunking, and Media Pipeline

1. Write high-risk contract tests for the new canonical attachment and directive surfaces before implementation lands, covering gateway rejection of malformed attachment payloads, bounded canonical attachment metadata, supported `[[reply:{external_message_id}]]`, `[[media:{url}]]`, and `[[voice]]` parsing, directive stripping from display text, malformed-directive failure behavior, and explicit prevention of raw directive text leaking to adapters.
2. Add additive migrations for append-only `message_attachments` and `outbound_deliveries`, including required enums, `retention_expires_at`, bounded provider metadata and error-detail fields, lookup indexes on `(message_id, ordinal)`, `(session_id, created_at)`, `(outbound_intent_id, chunk_index)`, and the unique `(outbound_intent_id, chunk_index)` constraint required for idempotent chunk retries.
3. Implement persistence models, repositories, and repository tests for append-only attachment and delivery records, covering terminal attachment states `stored`, `rejected`, and `failed`, idempotent lookup or insert behavior for attachment normalization retries, pending-row creation before adapter send, terminal `sent` or `failed` transitions, and duplicate chunk-key reuse without generating a second logical delivery attempt.
4. Extend `src/domain/schemas.py`, `apps/gateway/api/inbound.py`, `apps/gateway/deps.py`, and any related settings contracts so `POST /inbound/message` accepts optional canonical `attachments`, validates required `source_url` and `mime_type` fields plus bounded provider metadata, and preserves Spec 005 durable `202 Accepted` semantics without performing normalization inline.
5. Update `src/sessions/service.py` and `src/sessions/repository.py` so inbound acceptance persists canonical attachment references alongside the inbound message and queued run state for later worker-owned normalization, while keeping dedupe finalization, session resolution, and transcript-first durability unchanged.
6. Implement attachment normalization in `src/media/processor.py`, `src/context/service.py`, `src/jobs/service.py`, and supporting modules so the worker runs normalization as the first pre-context stage for inbound-triggered runs, enforces approved URL schemes plus MIME and byte-size allowlists, classifies `media_kind`, derives runtime-owned storage keys, computes `sha256`, records one terminal attachment row per input, and only exposes `stored` attachments to context assembly.
7. Add unit and integration coverage for attachment normalization and retry behavior, proving transient fetch or storage failures surface as retryable Spec 005 run failures, deterministic validation failures become terminal attachment rejections without unbounded run retries, resumed runs reuse terminal attachment state idempotently, and context assembly never consumes raw or non-terminal attachment payloads.
8. Implement shared reply-directive parsing and deterministic block chunking in `src/domain/reply_directives.py` and `src/domain/block_chunker.py`, with tests covering typed directive metadata extraction, capability-safe cleaned text output, deterministic paragraph-first chunking across capability profiles, hard-split fallback for overlong paragraphs, stable chunk ordering, and the guarantee that no empty chunks are emitted.
9. Extend runtime-owned outbound intent and execution wiring in `src/graphs/state.py`, `src/tools/messaging.py`, `src/channels/dispatch.py`, and `src/jobs/service.py` so the dispatcher combines persisted assistant display text with runtime-owned outbound intent metadata after the turn completes, performs reply-directive parsing, policy and capability checks, deterministic chunking, pending delivery-row creation, adapter sends, and terminal status persistence without moving transport logic into the graph or tools.
10. Implement adapter capability metadata and thin send interfaces in `src/channels/adapters/base.py`, `src/channels/adapters/webchat.py`, `src/channels/adapters/slack.py`, and `src/channels/adapters/telegram.py`, ensuring each adapter declares `max_text_chars` plus reply/media/voice support, translates inbound provider payloads into the canonical attachment contract where applicable, sends one text chunk or one media instruction at a time, and never invokes graph orchestration, parses directives, or owns chunking behavior.
11. Add dispatcher and adapter tests proving capability-gated fail-closed behavior for reply, media, and voice metadata, explicit failure recording when directives are unsupported or denied, stable append-only delivery auditing for chunked text and media sends, partial multi-chunk failure diagnosability, and contract enforcement that adapters cannot bypass the shared dispatcher or call graph orchestration directly.
12. Finish with end-to-end integration coverage for the supported `webchat`, `slack`, and `telegram` slice, proving inbound attachments preserve Spec 005 accept-and-queue semantics while normalization is deferred to worker execution, outbound delivery uses dispatcher-owned chunking instead of adapter-local logic, chunk retries remain idempotent by `(outbound_intent_id, chunk_index)`, text-only fallback remains available when media paths are disabled, and this phase implements post-turn chunked dispatch rather than true incremental transport streaming.
